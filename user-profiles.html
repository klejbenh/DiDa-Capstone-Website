Who This Backend Serves

Two users, two different reasons for wanting better sequencing.

These profiles explain who will use the platform, what they want from it, and how the backend should support their playlist analysis workflow.

User 1

Beginner DJ

DJ
Role of the user

A beginner DJ who wants help organizing songs into a smoother order so the playlist feels more intentional and professionally sequenced.

Goals of the user
  • Create playlists that sound more polished and DJ-like.
  • Improve transitions between tracks based on tempo, energy, and musical compatibility.
  • Learn which song order keeps momentum strong throughout a set.
Tasks of the user for the app
  • Log in with Spotify and choose a playlist to analyze.
  • Send playlist data to the backend for feature extraction and transition scoring.
  • Review the generated sequence and compare it to the original playlist order.
  • Use the reordered playlist as a starting point for practice sets or live performance prep.

User 2

Regular Music Listener

ML
Role of the user

A casual listener who wants playlists to feel seamless, uninterrupted, and enjoyable without needing DJ knowledge or manual reordering.

Goals of the user
  • Enjoy a more natural listening experience with fewer abrupt transitions.
  • Get playlists that match the mood and flow they want for everyday listening.
  • Trust the system to arrange songs in a way that feels smooth automatically.
Tasks of the user for the app
  • Log in with Spotify and select a favorite playlist or group of tracks.
  • Let the frontend pass that playlist data to the backend for analysis.
  • Receive an optimized playlist order designed for continuous listening.
  • Play the reordered sequence for workouts, studying, driving, or daily music sessions.
